   public class DragonCarmesi extends Thread {
       private HiloLance caballero;

       public DragonCarmesi(HiloLance caballero) {
           this.caballero = caballero;
       }

       @Override
       public void run() {
           while (caballero != null && caballero.getNivelChispa() < 100) {
               try { Thread.sleep(15000); } catch (InterruptedException e) {}

               System.out.println("DRAGON: El Dragon Carmesi cae en picado hacia Lance!");

               if (Math.random() < 0.40) {
                   System.out.println("LANCE: Alza su escudo y repele el fuego. +15 chispa!");
                   caballero.modificarChispa(15);
               } else {
                   System.out.println("DRAGON: Fuego! Lance sufre quemaduras. -30 chispa.");
                   caballero.modificarChispa(-30);
               }
           }
           System.out.println("DRAGON: El Dragon Carmesi huye hacia las montanas.");
       }
   }

   modificarChispa(int cantidad) accesibles publicamente. Si ya los tienes

ESCENARIO COMPAÑERO 3: SECUESTRO DE ELISABETHA + RESCATE DE LANCE
                       (Torre del Dragon con wait/notifyAll)
================================================================================

DESCRIPCION: Se crea una "Torre del Dragon" como nuevo recurso en el servidor.
Elisabetha puede ser secuestrada al pasear (se congela con wait()) y Lance
puede ir a rescatarla (la libera con notifyAll()). Usa comunicacion
por sockets y sincronizacion con monitores.

COMO HACERLO PASO A PASO:

--- PASO 1: Crear TorreDelDragon.java en Servidores/src/ ---

   Es un monitor (objeto compartido con synchronized) donde:
   - serSecuestrada() congela el hilo con wait()
   - rescatar() lo libera con notifyAll()

   public class TorreDelDragon {
       private boolean princesaSecuestrada = false;

       // Cuando Elisabetha es secuestrada, llama a esto
       public synchronized void serSecuestrada() {
           princesaSecuestrada = true;
           System.out.println("DRAGON: Un Dragon Carmesi ha secuestrado a Elisabetha!");

           // Mientras este secuestrada, EL HILO SE CONGELA (wait)
           while (princesaSecuestrada) {
               try {
                   wait();  // Congela el hilo que atiende a Elisabetha
               } catch (InterruptedException e) {}
           }
       }

       // Cuando Lance va a la torre, llama a esto
       public synchronized boolean rescatar() {
           if (princesaSecuestrada) {
               princesaSecuestrada = false;
               System.out.println("LANCE: Ha matado al Dragon y liberado a Elisabetha!");
               notifyAll();  // DESPIERTA el hilo congelado de Elisabetha
               return true;  // Hubo rescate
           }
           return false;  // La torre estaba vacia
       }
   }

   CLAVE: Esto es el patron PRODUCTOR-CONSUMIDOR / WAIT-NOTIFY que hemos
   visto en clase. El hilo de Elisabetha queda dormido en wait() hasta que
   Lance llama a notifyAll() desde rescatar().

--- PASO 2: Actualizar el Servidor para manejar los nuevos codigos ---

       if (Math.random() < 0.25) {  // 25% de probabilidad de dragon
           try {
               Socket socket = new Socket(HOST, PUERTO_TABERNA);
               DataOutputStream salida = new DataOutputStream(socket.getOutputStream());
               DataInputStream entrada = new DataInputStream(socket.getInputStream());

               System.out.println("[ELISABETHA] OH NO! Un Dragon la atrapa!");
               System.out.println("[ELISABETHA] Encerrada en la torre esperando rescate...");

               salida.writeInt(5);  // Codigo 5: Secuestro

               // Elisabetha se queda BLOQUEADA aqui hasta que Lance envie Codigo 6
               entrada.readInt();

   }

       System.out.println("[LANCE] Cabalga hacia las montanas buscando aventuras...");
       try { Thread.sleep(4000); } catch (InterruptedException e) {}

       try {
           Socket socket = new Socket(HOST, PUERTO_TABERNA);
           DataOutputStream salida = new DataOutputStream(socket.getOutputStream());
           DataInputStream entrada = new DataInputStream(socket.getInputStream());

           salida.writeInt(6);  // Codigo 6: Intento de rescate

           // El servidor responde true si Elisabetha estaba secuestrada
           boolean princesaRescatada = entrada.readBoolean();

           if (princesaRescatada) {
               System.out.println("[LANCE] Un Dragon Carmesi! Lucha y lo decapita!");
               System.out.println("[LANCE] Ha rescatado a la Princesa! +30 chispa.");
               modificarChispa(30);
           } else {
               System.out.println("[LANCE] La torre del dragon esta abandonada.");
           }

           socket.close();
       } catch (Exception e) {
           System.out.println("[LANCE] La montana es inaccesible (Fallo de red).");
       }
   }

   CLAVE: Lance envia codigo 6, el servidor ejecuta torre.rescatar() que
   hace notifyAll() si Elisabetha estaba secuestrada. Esto despierta

   a Elisabetha con writeInt(1) y ella se desbloquea tambien.

--- RESUMEN DEL FLUJO COMPLETO DEL SECUESTRO ---

   1. Elisabetha pasea por el bosque (25% probabilidad de dragon)
   2. Elisabetha conecta al servidor, envia codigo 5
   3. Servidor ejecuta torre.serSecuestrada() -> wait() -> HILO CONGELADO
   4. Elisabetha queda bloqueada en entrada.readInt() (el servidor no responde)
   5. [Mientras tanto, el tiempo pasa, otros hilos siguen ejecutandose...]
   6. Lance explora montanas, conecta al servidor, envia codigo 6
   7. Servidor ejecuta torre.rescatar() -> notifyAll() -> DESPIERTA HILO ELISABETHA
   8. Hilo de Elisabetha en servidor sale del while, ejecuta salida.writeInt(1)
   9. Elisabetha (cliente) recibe el readInt() y se desbloquea
  10. Ambos ganan +30 chispa

   PATRON USADO: Socket bloqueante + Monitor (synchronized/wait/notifyAll)
   = Exactamente lo que hemos visto en clase combinando ambas tecnicas.
